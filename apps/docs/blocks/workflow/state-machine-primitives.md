# Workflow ‚Äî State Machine Primitives

## Overview

The primitive layer of `blocks-workflow` is two types: `IWorkflowDefinition<TState, TTrigger, TContext>` (the immutable shape of a workflow) and `WorkflowInstance<TState, TTrigger, TContext>` (an immutable snapshot of a running instance). Everything else in the block ‚Äî the fluent builder, the runtime, the DI extension ‚Äî is built on top.

## Generic type contract

All workflow types are parameterised by three type arguments:

| Parameter | Constraint | Purpose |
|---|---|---|
| `TState` | `struct, Enum` | Enum representing workflow states. |
| `TTrigger` | `struct, Enum` | Enum representing workflow triggers (events or commands). |
| `TContext` | none | Arbitrary mutable context object carried per instance. |

The enum constraints on `TState` and `TTrigger` are enforced by the compiler. You cannot instantiate a workflow using, for example, `string` states ‚Äî the code will not compile. The context has no constraint, so you can use a class, a record, a tuple, or whatever carries the data you need.

## IWorkflowDefinition

```csharp
public interface IWorkflowDefinition<TState, TTrigger, TContext>
    where TState : struct, Enum
    where TTrigger : struct, Enum
{
    TState InitialState { get; }
    IReadOnlyCollection<TState> TerminalStates { get; }

    TState? Next(TState from, TTrigger trigger);

    ValueTask OnTransitionAsync(
        TState from,
        TState to,
        TTrigger trigger,
        TContext context,
        CancellationToken ct);
}
```

Members:

- **`InitialState`** ‚Äî the state assigned to new instances at creation.
- **`TerminalStates`** ‚Äî the set of states that represent completion. Instances in a terminal state reject further triggers.
- **`Next(from, trigger)`** ‚Äî returns the target state when the trigger is permitted from `from`, or `null` otherwise.
- **`OnTransitionAsync(‚Ä¶)`** ‚Äî optional side-effect hook invoked after a committed transition. The runtime awaits this on the caller's thread.

Definitions are designed to be built once, frozen, and shared across threads. The shipped implementation (`FrozenWorkflowDefinition<‚Ä¶>`) is internal; consumers obtain instances via `WorkflowDefinitionBuilder.Build()`.

## WorkflowInstance

```csharp
public sealed record WorkflowInstance<TState, TTrigger, TContext>(
    WorkflowInstanceId Id,
    TState CurrentState,
    TContext Context,
    bool IsTerminal,
    DateTimeOffset StartedAtUtc,
    DateTimeOffset LastTransitionAtUtc,
    int TransitionCount)
    where TState : struct, Enum
    where TTrigger : struct, Enum;
```

Snapshot of a running (or completed) instance. Each successful transition produces a new record ‚Äî the original is not mutated.

Fields:

- **`Id`** ‚Äî strong-typed `WorkflowInstanceId`.
- **`CurrentState`** ‚Äî the state the instance is in right now.
- **`Context`** ‚Äî the context object last written during a transition. Note: the context itself is referenced by the record and may be mutated by the transition hook; the record value is simply the reference captured at the point of the snapshot.
- **`IsTerminal`** ‚Äî convenience flag set when `CurrentState` is in the definition's `TerminalStates`.
- **`StartedAtUtc`** ‚Äî when `StartAsync` was called.
- **`LastTransitionAtUtc`** ‚Äî UTC of the most recent state change; equals `StartedAtUtc` before any triggers have been fired.
- **`TransitionCount`** ‚Äî number of successful state transitions since start.

## FrozenWorkflowDefinition

`FrozenWorkflowDefinition<TState, TTrigger, TContext>` is the internal implementation returned by `WorkflowDefinitionBuilder.Build()`. It is explicitly immutable and thread-safe:

- `_transitions` is captured as an `IReadOnlyDictionary` keyed by `(TState From, TTrigger On)`.
- `_terminalStates` is captured as a `HashSet` at build time.
- `_onTransition` is either `null` or a captured delegate; calling `OnTransitionAsync` is a straightforward null-check-then-invoke.

`Next(from, trigger)` is a `TryGetValue` against `_transitions` ‚Äî O(1). Consumers can evaluate "is this trigger allowed?" cheaply enough to drive UI affordances (enable/disable buttons) directly off the definition.

## Instance identity

`WorkflowInstanceId` is a strong-typed id struct generated by `WorkflowInstanceId.NewId()`. It is the key used by the runtime to look up stored instances.
