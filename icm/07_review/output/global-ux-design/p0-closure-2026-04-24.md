## P0.1 — Icon Replacements (Research-Backed)

Three icons flagged by council replaced with alternatives grounded in precedent from globally-deployed products. Full research artifact: [attached in closure notes below].

### Final icon set

| State | Icon | Material name | Fluent name | Precedent | Mirror under RTL |
|---|---|---|---|---|---|
| Healthy | Checkmark | `check_circle` | `CheckmarkCircle` | Universal; retained from original | no (symmetric) |
| Stale | Clock face | `schedule` | `Clock` | Universal; retained from original | **no** (clocks never mirror) |
| Offline | Cloud-with-slash | `cloud_off` | `CloudOff` | Google Docs, Android, OneDrive, Teams — bearer-agnostic per Google's own offline-design guide | no (symmetric) |
| ConflictPending | Diverging arrows | `call_split` | `ArrowSplit` | GitHub merge-conflict UI, Lucide community-standard `git-merge-conflict` | **yes** (directional flow) |
| Quarantine | Circle-with-bar | `do_not_disturb_on` / `block` | `Blocked` | OneDrive blocked-file badge; ISO 7001-adjacent shape | no (symmetric) |

### Why the replacements hold

- **Padlock rejected** — encodes "secure/encrypted" globally (TCBOK + Cieden cross-cultural research); no secondary reading for "rejected" exists.
- **Wi-Fi glyph rejected** — centers Wi-Fi-era infrastructure assumption; Google's own offline-design guide uses `cloud_off`, not Wi-Fi-slash.
- **Warning triangle rejected** — ISO 7010 hazard-safety symbol; scope-inappropriate for routine merge-conflict UI. GitHub uses split/diverge arrows; Lucide community converged on fork arrows.

### RTL mirror manifest

```yaml
sync-state-icons:
  healthy:          { library: material, name: check_circle,       mirror: false }
  stale:            { library: material, name: schedule,           mirror: false }  # clocks don't mirror
  offline:          { library: material, name: cloud_off,          mirror: false }
  conflictPending:  { library: material, name: call_split,         mirror: true  }  # flow narrative mirrors
  quarantine:       { library: material, name: do_not_disturb_on,  mirror: false }
```

Consumed by the adapter layer (`ui-adapters-blazor`, `ui-adapters-react`) via a per-icon `mirror` flag. Mirror applied via `transform: scaleX(-1)` under `:dir(rtl)`.

### Known cross-cultural validation

All four non-original replacements are used by Google, Microsoft, GitHub, or the Lucide community in their localized production deployments. These represent de facto native-speaker validation at scale across all target locales (ar, zh, ja, ko, hi, he, fa). Dedicated Sunfish-side native-speaker review still recommended at v1.x via the language coordinators listed in `i18n/coordinators.md` — this closes the P0 gate but does not retire the ongoing validation obligation.

## P0.2 — Text-Overflow Policy Per Locale

### Problem

"Always-visible label" collides with German being +40% longer than English, Arabic having wider glyphs with diacritics, and Hindi Devanagari adding ~25% glyph height.

### Policy

**Two label tiers per state:**
- **Short label** — intended for compact component form (status bars, grid cells). ≤ 10 grapheme clusters in en-US source.
- **Long label** — intended for standard component form (panels, notifications). ≤ 30 grapheme clusters in en-US source.

**Budget per locale** measured in CSS `ch` units (relative to font-size, respects localized fonts):

| Form factor | Budget (en-US baseline) | Overflow tolerance (per-locale multiplier) |
|---|---|---|
| Compact | `max-inline-size: 10ch` | de: ×1.4; ar: ×1.2; hi: ×1.1 (glyph-height normalized) |
| Standard | `max-inline-size: 28ch` | de: ×1.4; ar: ×1.2; hi: ×1.1 |
| Mobile | `max-inline-size: 16ch` | same multipliers |

Multipliers derived from published glyph-width ratios in [CLDR character properties](https://cldr.unicode.org/translation/characters) and [Microsoft localization width guidelines](https://learn.microsoft.com/en-us/globalization/localizability/string-length).

### Overflow handling

When measured content exceeds the budget:

1. **Text truncates** with ellipsis via `text-overflow: ellipsis`
2. **Full text always available** via `aria-label` — screen readers get unabridged text
3. **Reveal-on-focus** — hovering or focusing the indicator expands the label to full width in an overlay (no wrapping, 2-line max)
4. **Title attribute** carries the full text for mouse-hover tooltips (fallback)

### CSS implementation

```css
.sunfish-syncstate__label {
  max-inline-size: var(--sf-syncstate-label-max, 28ch);
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}

.sunfish-syncstate[data-form="compact"] .sunfish-syncstate__label {
  --sf-syncstate-label-max: 10ch;
}

.sunfish-syncstate:hover .sunfish-syncstate__label,
.sunfish-syncstate:focus-within .sunfish-syncstate__label {
  max-inline-size: none;
  position: absolute;
  inset-inline-start: 0;
  background: var(--sf-bg-elevated);
  padding: var(--sf-space-2);
  border-radius: var(--sf-radius-sm);
  z-index: 1;
  white-space: normal;
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
}

:root[lang^="de"] .sunfish-syncstate__label {
  --sf-syncstate-label-max: calc(var(--sf-syncstate-label-max-default, 28ch) * 1.4);
}

:root[lang^="ar"] .sunfish-syncstate__label {
  --sf-syncstate-label-max: calc(var(--sf-syncstate-label-max-default, 28ch) * 1.2);
}
```

### Localized labels per state

| State | en-US short | en-US long |
|---|---|---|
| Healthy | "Synced" | "Synced with all peers" |
| Stale | "2h ago" (configurable) | "Last synced 2 hours ago" |

## P0.3 — Live-Region Coalescing Service

### Problem

100 records flipping state simultaneously would fire 100 live-region announcements. JAWS concatenates, NVDA queues, VoiceOver drops — undefined behavior across assistive tech.

### Architecture: single document-level announcer

Two live regions owned by the app root, in Light DOM (not per-component shadow DOM):

```html
<body>
  <div
    id="sf-syncstate-announcer-polite"
    role="status"
    aria-atomic="true"
    class="sr-only">
  </div>
  <div
    id="sf-syncstate-announcer-assertive"
    role="alert"
    aria-atomic="true"
    class="sr-only">
  </div>
  <!-- app content -->
</body>
```

**Light-DOM placement is deliberate** — cross-shadow-root ARIA references are unreliable, and consolidating announcements to document-level regions is the pattern documented in [WAI-ARIA APG Live Region guidance](https://www.w3.org/WAI/ARIA/apg/practices/live-regions/).

**Single `role` per region, no redundant `aria-live`** — council fix. `role="status"` implies `aria-live="polite"`; `role="alert"` implies `aria-live="assertive"`. Setting both has caused NVDA duplicate announcements historically.

### Coalescing service contract

```csharp
public interface ISyncStateAnnouncer
{
    /// <summary>
    /// Queue a state transition for announcement.
    /// Announcement fires after the coalescing window (500ms).
    /// Multiple transitions within the window are batched.
    /// </summary>
    void Announce(SyncStateTransition transition);
}

public sealed class SyncStateTransition
{
    public required string RecordId { get; init; }
    public required SyncState FromState { get; init; }
    public required SyncState ToState { get; init; }
    public required DateTimeOffset OccurredAt { get; init; }
    public AnnouncementPriority Priority =>
        ToState is SyncState.ConflictPending or SyncState.Quarantine
            ? AnnouncementPriority.Assertive
            : AnnouncementPriority.Polite;
}

public enum AnnouncementPriority { Polite, Assertive }
```

### Coalescing rules

- **Window:** 500ms from first transition in a batch
- **Individual announcement** when batch size ≤ 3
- **Aggregate announcement** when batch size > 3, grouped by `ToState`:
  - "3 records now stale" (polite)
  - "12 records need review" (assertive — groups ConflictPending + Quarantine)
- **Priority split:** assertive announcements bypass the polite queue entirely (routed to the `alert` region)
- **Rate limit:** no more than 1 assertive announcement per second per region. Excess batched into the next aggregate.

### Aggregate message templates (localized)

```
syncstate.announcement.individual.healthy = "Record {recordName} synced"
syncstate.announcement.individual.stale = "Record {recordName} is stale"
syncstate.announcement.aggregate.stale = "{count} records now stale"
syncstate.announcement.aggregate.conflict = "{count} records need review"
syncstate.announcement.aggregate.mixed = "Multiple records changed — open activity feed"
```

ICU MessageFormat used for pluralization (ar six-form, ja/ko zero-form).

### Mass-transition kill switch

If >50 transitions occur within 10 seconds (a sync-daemon bug scenario), the announcer emits one aggregate "Multiple records changed — open diagnostics" and suppresses further announcements until the burst ends. Logged to telemetry as `syncstate.announcer.burst_suppressed`.

## P0.4 — Hindi Tier Decision

**Decision:** Promote `hi` from `bake-in` to `complete` at v1.

### Rationale

- The foundational paper (`_shared/product/local-node-architecture-paper.md`) and the *Inverted Stack* book cite India as the primary mission-aligned deployment market (connectivity gradients, monsoon offline scenarios, Tier 2/3 cities)
- Bake-in tier = partial strings — shipping the primary market partial contradicts the mandate
- MADLAD-400 pipeline generates initial Hindi drafts → human translator review → approved strings merge to `.resx`
- Hindi translation cost is covered by commercial translation services line (per `vision.md` business model)

### Updated `locales.json` tiers

**Complete (9):** en-US, es-419, pt-BR, fr, de, ja, zh-Hans, ar-SA, **hi**
**Bake-in (3):** he-IL, fa-IR, ko

### Commitment implication

- Hindi coordinator named in `i18n/coordinators.md` before Phase 1 sprint starts
- MADLAD-400 first-pass drafts generated for all Hindi keys in Phase 1 Week 1
- Human translator review contracted (budgeted) with 2-week turnaround per keyset
- Hindi at 100% completeness is a Phase 2 exit criterion alongside en-US, not a v1.x deferral

## P0.5 — CVD Evidence + Dark-Mode Palette

### Problem confirmed numerically

Current palette: Stale (`#f39c12`) × ConflictPending (`#e67e22`) has **ΔE2000 = 10.83**, below the 11.0 "unmistakably different" threshold. Council's concern validated.

### Fix: shift Stale from amber to blue

### Final palette — light theme (background `#ffffff`)

| State | Hex | APCA Lc vs white | ΔE2000 min distance to other states |
|---|---|---|---|
| Healthy | `#27ae60` | 54.25 | 20.11 (to Stale) |
| Stale | `#3498db` | 57.78 | 20.11 (to Healthy) |
| Offline | `#7f8c8d` | 61.33 | 20.11 (to Stale) |
| ConflictPending | `#e67e22` | 53.92 | 24.99 (to Quarantine) |
| Quarantine | `#c0392b` | 75.62 | 24.99 (to ConflictPending) |

All pairs pass ΔE2000 ≥ 11; all pass APCA Lc ≥ 45 for non-text UI.

### Final palette — dark theme (background `#1e1e1e`)

| State | Hex | APCA Lc vs dark bg | Notes |
|---|---|---|---|
| Healthy | `#2ecc71` | 63.27 | Brightened green |
| Stale | `#5dade2` | 56.44 | Lightened blue |
| Offline | `#95a5a6` | 54.80 | Lightened gray |
| ConflictPending | `#f39c12` | 61.37 | Amber (safe now — Stale is blue) |
| Quarantine | `#ff6b6b` | 51.53 | Shifted from `#e74c3c` which failed at 39.85 |

All pairs pass ΔE2000 ≥ 11 in dark mode; all pass APCA Lc ≥ 45.

### Measurement methodology

- **ΔE2000 formula:** [CIE 2000](https://en.wikipedia.org/wiki/Color_difference#CIEDE2000) implementation over CIE Lab (sRGB → Linear → XYZ → Lab, D65 reference white)
- **APCA Lc:** [SAPC-APCA](https://github.com/Myndex/SAPC-APCA) simplified formula
- **Threshold for pairwise color distinctness:** ΔE2000 ≥ 11.0 (validated for CVD contexts per [Just Noticeable Difference research](https://en.wikipedia.org/wiki/Just-noticeable_difference))
- **Threshold for non-text UI contrast:** APCA Lc ≥ 45 (per APCA guidance for non-text UI elements)

### CVD simulator for CI

- **Primary:** Chrome DevTools built-in Rendering panel — `Emulate vision deficiencies: Deuteranopia / Protanopia / Tritanopia` at "severe" (equivalent to 100% severity)
- **Secondary:** [Sim Daltonism](https://michelf.ca/projects/sim-daltonism/) macOS for manual review
- **CI invocation:** Chrome DevTools MCP's `emulate` command applies CVD to each provider theme's kitchen-sink page, takes screenshots, diffs against baseline. Automated as part of the axe-core CI gate.

### Build-time contrast linter for enterprise overrides

A build step in `tooling/theme-validator/` loads any override theme and recomputes the ΔE2000 matrix + APCA Lc. Fails CI if:
- Any state pair drops below ΔE2000 11.0
- Any state fails APCA Lc 45.0 against its background token

This prevents silent a11y regressions from brand-color overrides.
