# Intake — ADR 0028 Amendment A10 (mechanical retraction): A6.1 `schemaEpoch` citation correction

**Date:** 2026-05-01
**Requestor:** XO (declared by ADR 0028-A9.8.4 as the parent-amendment retraction declared by the A9 council finding F1)
**Request:** Amend ADR 0028 with a new amendment **A10** that retracts the structurally-incorrect citation in A6.1 + the A1 envelope contract: both cite "ADR 0001 schema-registry-governance" for the `schemaEpoch` semantic, but ADR 0001 does not define `schemaEpoch`. ## Problem Statement

ADR 0028's A6.1 (the version-vector tuple type signature definition; landed via PR #395) defines:

```text
schemaEpoch:   uint32,                  # monotonic per-schema-cutover; per ADR 0001 schema-registry-governance
```

The "per ADR 0001 schema-registry-governance" comment is **structurally incorrect**. Verified via `git show origin/main:docs/adrs/0001-schema-registry-governance.md | grep -i epoch` returning **zero matches**. ADR 0001 governs schema-registry tier governance (repo-local vs federated namespace), NOT epoch coordination.

The `schemaEpoch` semantic IS defined — but in the **paper**, not in any ADR:
- **Paper §7.1 (Expand-Contract Pattern)** line 220: *"This is a breaking change requiring a **schema epoch bump** — a version gate that rejects sync connections from nodes below the minimum supported epoch."*
- **Paper §7.4 (Epoch Coordination and Copy-Transform Migration)** line 238: *"For truly breaking changes, the architecture uses **schema epochs** coordinated by distributed lease quorum"* + *"Schema epoch coordination protocol"*

This citation failure was caught by the **ADR 0028-A9 council** (PR #433; in-thread XO authored 2026-05-01) as F1 Critical structural-citation finding. The A9 derivative amendment fixed its own citation in A9.8.1 but explicitly declared the parent A6.1 retraction as XO follow-up — that's this intake.

## Cohort lesson

The A9 council's F1 finding established a new failure mode: **parent-amendment citation propagation into derivative amendments via verbatim copy.** When a derivative amendment cites the parent amendment's text, structural-citation verification must extend to the parent's own citations — not just confirmation that the parent ADR is Accepted. The §A0 self-audit pattern caught zero of A9's parent-propagated structural-citation failures (consistent with ADR 0063-A1.15 lesson that §A0 is necessary but not sufficient).
