## Non-obvious outcomes

- **Spec v0.3 → v0.4 pivot was load-bearing.** The Iced lessons-learned (UI-toolkit research) drove the multi-platform-host posture decision. v0.3 had assumed a more Blazor-centric shape; v0.4 generalized.
- **Pattern C contractor-portal macaroon** (Phase D-9, PR #16). First production use of the macaroon capability model that later anchors ADR 0032 (multi-team Anchor) + ADR 0046 (key-loss recovery). Macaroon pattern proved out before the formal ADR.
- **Postgres asset-store + IPFS blob-store** (PRs #15, #17). Two persistence backends shipped in same week. Established the early provider-neutrality pattern that later got formalized in ADR 0013 + analyzer SUNFISH_PROVNEUT_001.
- **Atomic batch entity creation** (G21, PR #32) and **streaming blob-write path** (G22, PR #31). Both shipped as in-memory-first-pass with the persistence-tier follow-up deferred. This pattern (substrate first; persistence adapter later) becomes the cluster-wide convention.
- **JsonConverter flatten + transitive Blazor-dep guard** (G36, PR #27). Caught a transitive Blazor dependency leaking into a foundation-tier package; analyzer added to prevent recurrence. Small fix; large cumulative impact (foundation packages stay framework-agnostic).
- **Parking-lot sweep** (PR #33). Resolved G12, G13, G23, G24, G25, G26, G29, G30, G35 docs-only. First instance of "batch-resolve docs-only items in one PR" pattern that later becomes a fallback-ladder rung.
